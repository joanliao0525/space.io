說在前面

為讓彼此成為更好的人，請不吝嗇給予彼此真實的回饋與讚美，針對專案分組開始到結束，人、事、物皆可回饋。

小菜回饋

專案

起初發想夢很大，覺得需要加入打卡、休假功能、勞健保試算…等，討論了一個禮拜多，當時進度是六組中最慢的，隨著時間遞減，壓力也越大，最後討論出以「打工薪資試算」為主。

問題點

  1. 花太多時間討論，進度停滯一週左右
  2. 功能比預期少，略薄弱

解決方案

  1. 直接分票開始做會更有效率
  2. 依團隊 member 能力調整，當無法做更多功能時，策略改以行銷、包裝為主

給團隊成員的話

起初大家對於 Rails、Git 不熟，在混亂中成長，隨著每天使用，逼自己成長時，Git 操作有比當初還清楚，對於需要完成的工作內容或交辦事項，也是一種磨合，沒人喜歡被管或限制，此時自律非常的重要，能否成為讓人放心交託，這時便知曉。

在軟體工程師的路上，門檻確實比一般產業高許多，不論是技術或細心皆是，打錯一個字或少個標點符號，都會造成系統錯誤，未來工作時不會有人在一旁隨時提醒，在這條道路上，自我解決問題的能力比會那些技術來的更重要，期許大家都能成為獨當一面的工程師 :+1:

問題點

  1. push GitHub 後，須自行檢查 Files changed 改了哪些檔案，沒人有義務幫你檢查
  2. 養成主動和大家說發 pr 並描述這次 pr 做了什麼事情
  3. 主動回報進度及預計完成時間
  4. 每個人都要學會部署，改完的地方先自行部署測試，確認沒問題才發 pr (本地端測試 ok 不等於部署上去也 ok ，學會就是你的)
  5. 修復完的 bug 務必自行測試過，並檢查原本功能是否好的變壞的
  6. 養成自行解決問題的能力，而非請助教當打手，工作時要自己面對，該學的是助教如何解決問題，而非要答案
  7. 養成主動找出問題、解決問題 (像是功能是否壞掉、RWD、多國語言下是否跑版等問題，想像成你就是使用者、評審，盡可能玩壞它並回報問題)
  8. 討論正式時，Line 上面不建議丟貼圖
  9. 討論時，找團隊成員一起開會，而非搞小團體，空間這麼小，看到會不會說而已 (大忌)

友渝回饋

專案

這次的專案後端大部分都是小菜負責，我這邊除了 FactoryBot 之外就沒有幫忙到後端的事情，並且對於UI設計還有待加強，各方面都要繼續加油，這次的專案辛苦大家的努力付出，希望大家都對這個專案的成果滿意以及認識自己的不足，大家一起加油，在 coding 的路上繼續努力！

問題點

後端方面還需要再努力精進，前端的UI設計不夠成熟、考慮的不夠全面，對於原文文件的閱讀也需要加強。

解決方案

後端目前在複習上課影片並操作，前端打算多練習切版和 UI 以外，加強學習 JS ，並且增加練習。

宥誠回饋

專案

自己在這次專案，沒有幫到什麼忙，辛苦小菜及各位carry，大家都辛苦了，就像評審分享的，自己還有很多不足的地方，要繼續加油，尤其小菜在這次專案，大家都知道你付出了很多辛苦了

問題點

對於專案的Code熟悉度不夠，看不懂其他人得Code，目前自己的能力較不足還許要彌補，pdf的gem部署上heroku的問題也是很多，未來再面對套件要研究更徹底些，git 版控有因為這次專案更了解了，對於助教方面知道自己的不足虛心受教會再努力

解決方案

目前正在補足Rails的基本架構及用法，還有去研究其他人的code，彌補自己的不足

小派回饋

專案

剛開始沒參與到大家的討論，但已決定參加專案，決定大家想作主題後就全力配合，只希望自己別拖累大家就好

問題點

  1. 低估白天工作繁重程度，導致團隊討論時間無法投入，另外camp補課課程也delay
  2. i18n的跑版問題讓大家最後費心調整

解決方案

  1. 評估自己接案要更謹慎，時間規劃要更清楚，以能完整參與團隊討論為優先，否則就專心補完課程
  2. i18n其實不難，可以一剛開始納入決定是否要做，也可以讓前後端在命名和排版更快速，有多的時間讓成員多做更多功能

給團隊成員

雖然是目標前後端工程師，但前置作業的UX到UI的處理也要試著熟悉，也多看更多UI作品，好決定版面設計方向和溝通好命名原則，這樣在分工前端頁面處理，在推上去整合時更快更不容易有衝突

六角校長 回饋

  1. 都要考慮使用者情境(手機、Pad、PC)
  2. 提供測試帳號 (1、4)
  3. 表單防呆 * 號、icon 辨識
  4. 正式資料，不會認為是假網站

第一組 Oh my wage(內部系統)

  1. 建議提供測試帳號
  2. 職員要新增時薪時數時，新增按鈕要放在上頭
  3. 關鍵字搜尋 (placeholder 上可以有些提示內容)
  4. 新增哈利波特，裡面就有資料
  5. NTD 有薪資時，不應該刪除，而是結清後才可以踢除。
  6. 表單有系統
