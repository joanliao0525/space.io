Travis CI 介紹

Travis CI 是在軟體開發領域中的一個在線的，分布式的持續集成服務，用來構建及測試在 GitHub 託管的代碼。這個軟體的代碼同時也是開源的，可以在 GitHub 上下載到，儘管開發者當前並不推薦在閉源項目中單獨使用它。
它提供了多種程式語言的支持，包括 Ruby、JavaScript、Java、Scala、PHP、Haskell 和 Erlang 在內的多種語言。許多知名的開源項目使用它來在每次提交的時候進行構建測試，比如 Ruby on Rails，Ruby 和 Node.js。 — Wikipedia

Travis CI 設定完畢可以在使用者將程式碼上傳至 GitHub 時自動執行測試，包含使用者自己在其專案中寫的功能測試也可以在上傳時自動執行測試喔！
Travis CI 有分兩個網站，分別是 travis-ci.org 與 travis-ci.com，
travis-ci.org 可存取 public repository，若想存取 private repository 則需使用 travis-ci.com，但是兩個網站的設定步驟差不多，所以都可以參考喔！

網頁設定

使用 GitHub 帳戶登入

首次登入需要同意授權功能及存取資料

同意授權後會需要輸入一次 GitHub 的密碼

隨後就會到以下頁面，接下來就直接前往 GitHub 頁面設定囉！

右上角的頭像戳下去，點選 Settings

左邊選項分類清單點選 Applications，就會看到 Travis CI 並且點選他旁邊的 Configure

在最下方可以選擇是要將全部的 repository 授權給 Travis CI 或是只授權部分 repository

若是只授權部分，可以在這邊選擇要授權的 repository，日後想修改也都是可以的喔！

設定完後會自動導向 Travis CI 的頁面

以下有 travis-ci.com 和 travis-ci.org 兩個網站的設定頁面，可以看到 travis-ci.com 只看得到 GitHub 同意授權的 repository，travis-ci.org 則是看得到全部的 repository ，並且可以在此頁面中設定是否開啟測試功能

travis-ci.com

travis-ci.org

在專案設定頁面的 Webhooks 中點選 Edit 可以做授權的細項設定，也可以看到最近傳送給 Travis CI 的測試封包紀錄及測試結果，會在封包名稱旁有一個小符號，如果是綠色勾勾就代表測試通過

網頁設定到這邊就完成囉，接下來就是裝置設定！

裝置設定

將 Travis CI gem 加入 Gemfile 中，可以去 rubygem.org 找最新版本

存檔後執行$bundle

接下來在終端機中登入 Travis CI，下方終端機指令中的 --pro 只有當使用者授權的 Travis CI 網站為 travis-ci.com 時才會使用到，因為終端機預設的網站為 travis-ci.org，不是 travis-ci.com，所以會出現登入失敗的情況，這時候只要輸入 $travis login --pro 再登入一次就可以了

成功登入之後就可以使用以下指令來產生 .travis.yml 檔案

在 .tavis.yml 檔案中可輸入使用者所用的程式語言、資料庫類型及預設執行的指令，我這裡使用的是 Ruby on Rails 的專案，所以設定內容也是和專案內容相關的，官網上有文件可以更詳細的參考其他程式語言或資料庫的設定喔！

詳細設定文件在這裡 ( travis-ci.org & travis-ci.com 通用 ) ： Travis CI Docs

下方程式碼中資料庫的名稱和 database.yml 檔案中的是一樣的

設定完後就可以 git add .、git commit -m "commit name"、git push 上傳到 GitHub 線上 repository ，接著到 GitHub 頁面查看剛上傳的 commit 點有沒有開始跑測試，如果開始測試，會有一個黃色點點，測試成功會變成綠色勾勾，失敗則是紅色叉叉，如果有順利出現測試符號就代表成功啦！

在發 PR 時下方的 merge 按鈕就會變成這樣，下方也可以查看在這個 PR 當中每一個 commit 的測試結果是否成功
